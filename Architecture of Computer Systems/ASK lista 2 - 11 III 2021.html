Żeby prawe wyrażenie obliczyło się do fałszu 3ci od prawej bit w x musi być równy 0, ale żeby lewa strona obliczyła się przy tym do fałszu 3ci bit od prawej w x musi być równy 1. Dochodzimy więc do sprzeczności -> zawsze prawda.

(x * x) >= 0

Dla wszystkich takie liczby, dla których mnożenie spowoduje zapalenie się pierwszego bitu (MSB).

Jedyna taka sytuacja może zaistnieć kiedy będziemy mieć 0, no ale ono jest niemożliwe do osiągnięcia bo zanegowanie -1 da 000…1 -> to zawsze będzie prawda.

int32_t threefourths( int32_t x ){
    int32_t pom = x >> 2; //dzielenie całkowite
    int32_t r = x & 3; //reszta z dzielenia przez 4
    pom = pom + ( pom << 1 ); // pom *= 3
    r = r + ( r << 1 ); //r *= 3
    return pom + ( r >> 2 );
}

Najpierw w zmiennej pom przechowujemy wartość dzielenia całkowitego liczby x przez 4, w zmiennej r resztę z dzielenia przez 4. Mnożymy przez 3 resztę i podłogę z dzielenia przez 4 a następnie dodajemy część całkowitą tej reszty z dzielenia przez 4 (dokładamy tyle czwórek, ile zmieści się w reszcie).

int32_t compareLess( int32_t x, int32_t y ){
    return ( ( x >> 1 ) - ( y >> 1 ) - ( ~x & y & 1 ) ) >> 31 & 1;
}

Funkcja ma zwrócić 1 jeśli x jest mniejsze od y, 0 wpp.
Przesuwamy x i y w prawo, żeby uniknąć wyjścia poza zakres, wynik odejmowania tych dwóch liczb będzie liczbą ujemną (dodatnią wpp lub równą ( co do wszystkich bitów poza ostatnim ) 0).
Wynik k = ( ~x & y & 1 ) dla poszczególnych wartości ostatniego bita:
x = 0, y = 0 -> 0
x = 1, y = 0 -> 0 ( wtedy X jest większy )
x = 1, y = 1 -> 0
x = 0, y = 1 -> 1 ( wtedy Y jest większy ).
Kiedy Y jest większe od x to ( x >> 1 ) - ( y >> 1 ) < 0, czyli jak to przesuniemy
Zatem kiedy liczby są równe z dokładnością co do LSB, to dostaniemy 0 - wynik porównania k >> 31 & 1 ( czyli 0 gdy X jest większy lub równy Y i 1 gdy Y jest większy ).

Zadanie 7





int32_t abs( int32_t x ){
    int32_t m = x >> (N - 1);
    return ( x ^ m ) - m;
}

Bierzemy taką maskę x >> N - 1 która jest albo zerem (w zapisie binarnym same 0) dla liczby dodatniej albo -1 dla liczb ujemnych (w zapisie binarnym same 1). Czyli wykonując XORa naszego x z samymi 1kami dostaniemy negację tej liczby ( 0 ^ 1 -> 1 i 1 ^ 1 -> 0 ) a z samymi 0 nic się nie zmieni ( 1 ^ 0 -> 1 i 0 ^ 0 -> 0 ).
Zatem dla liczby dodatniej będziemy mieć (x ^ 0) - 0 = x - 0 = x.
A dla liczby ujemnej (x ^ (-1) ) - (-1) = !x + 1 no a to jest po prostu ta liczba z ujemnym znakiem.

Zadanie 8




int32_t sgn( int32_t x ){
    return x >> (N - 1) | -x >> (N - 1) & 1;
}

Lewa strona ORa obliczy się do 0 w przypadku liczb nieujemnych i do -1 kiedy liczba jest ujemna. Druga strona:
Lewa strona ANDa obliczy się do 0 gdy liczba jest niedodatnia i do -1 gdy jest dodatnia. Kiedy zrobimy na niej ANDa to gdy liczba jest niedodatnia dostaniemy 0 (0&1 = 0) a gdy dodatnia to 1 ( -1 & 1 = 1 ).
Zatem:
Gdy x = 0: 0 | 0 -> 0
x > 0: 0 | 1 -> 1
x < 0: -1 | 0 -> -1.

}
